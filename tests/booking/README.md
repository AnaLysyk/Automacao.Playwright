# [REGRESSAO] / [ASSISTIDO] Booking

## Objetivo

Agrupar testes do Booking/agendamento consumidos pelo Cidadao Smart.

## Tipo de execucao

Misto. `public` tende a regressao; `manual-assisted` depende de QA.

## Comando principal

```bash
npm run test:booking:public
```

Assistido:

```bash
npm run test:booking:assistido
```

## Pode entrar aqui

- Agendamento publico.
- Resumo, confirmacao e autenticacao do Booking.
- Fluxos assistidos conectados ao Booking.

## Nao pode entrar aqui

- POC tecnica isolada.
- Teste legado.
- Credenciais ou evidencias locais.
